# Signify — Roadmap

## Now

Ship to production: enable GitHub Pages, deploy to the 5 users via M365 Admin Center, and test end-to-end across OWA, New Outlook, and classic Outlook desktop. _(The WYSIWYG editor shipped in v1.2.0.)_

## Next

- **Easy no-backend image hosting** so users can add a logo that displays reliably everywhere without finding their own host (see `02_BACKLOG.md` / `04_DECISIONS.md`)
- Replace `window.prompt` URL entry with an inline in-pane input field

## Later

- Per-template starting points beyond the single starter layout (e.g. role-specific compliance starters)
- Multi-signature support (primary + abbreviated version for internal emails)

## Gated

- **Mobile support** — Outlook mobile does not support task pane add-ins; no path forward currently
- **Centrally managed templates** — would require a backend server; out of scope for a free self-hosted solution
